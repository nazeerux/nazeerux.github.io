UX Case Study

e-Challan Portal – Hero Section Redesign (Concept)


Role Product / UX Designer Platform Web (Desktop-first, responsive-ready) Tools Figma Timeline 2–3 focused design sessions
1. Project Overview
This project explores a concept redesign of the hero section for a government-style e-Challan portal. The hero section is often the most critical point of entry, where users decide whether they trust the platform enough to proceed—especially when payments are involved.
The redesign focuses on clarity, trust, and ease of action, while maintaining an institutional and official tone appropriate for public-service interfaces. ⚠️ This is a concept project created for portfolio purposes.

2. Problem Statement

Many public-service portals struggle with: Poor visual hierarchy Overloaded content in the first screen Low trust perception for payment-related actions Unclear primary call-to-action As a result, users feel hesitant, confused, or overwhelmed within the first few seconds of landing on the page.

3. Design Goals

Clearly communicate what the service does within seconds Guide users toward a single primary action Establish trust without heavy or distracting elements Reduce cognitive load for first-time users Maintain a calm, government-appropriate visual language

4. User Context & Assumptions

Primary Users Vehicle owners checking challan status Users paying traffic fines online First-time users unfamiliar with the process User Needs Quick understanding Confidence in legitimacy Simple, guided next steps Because this is a concept project, insights are based on heuristic evaluation and patterns commonly seen in public-sector UX.

5. UX Strategy & Design Decisions


a. Visual HierarchyThe hero section was structured to guide attention in this order:
Headline (what the service is) Primary CTA (what to do next) Supporting description Trust indicators Illustration This ensures users understand the purpose before being asked to take action.

b. Content Simplification
Reduced text to essentials only Avoided technical jargon Used instructional, plain language This allows users to scan and act quickly.

c. Trust & Credibility Signals
To reduce hesitation: Included short trust statements (secure payments, official service, receipts) Added usage/coverage indicators to reinforce legitimacy Avoided emojis or decorative elements to keep tone formal Trust cues were kept subtle, not promotional.

d. Illustration Treatment
Used a contextual illustration related to traffic enforcement Reduced illustration opacity to 85% Ensured text and CTA always dominate visually The illustration supports understanding without competing for attention.

e. Color & Depth
Chose a blue and white palette to communicate trust, calm, and reliability Used white space to improve readability Added subtle depth using HSL-based lightness and saturation adjustments instead of heavy shadows This created a soft 3D feel while keeping the interface clean and professional.

f. Grid & Spacing System
Applied a consistent column grid Used an 8px spacing system for layout rhythm Ensured clean alignment across text, CTAs, and stats This improves scannability and developer handoff.

g. Accessibility Considerations
Adequate color contrast Readable font sizes and line heights Clear button labels No reliance on color alone to convey meaning

7. Final Outcome
The redesigned hero section: Communicates purpose within seconds Feels trustworthy and official Reduces cognitive load Guides users toward action clearly Aligns with real-world government UI standards

8. Learnings & Reflections
In public-service UX, restraint matters more than visual flair Trust signals are as important as functional clarity A strong hero section can significantly improve user confidence Small details (spacing, opacity, hierarchy) have large UX impact If extended further, this project could include: Full user flow for challan checking and payment Error states and edge cases Mobile-first optimization Usability testing with real users

9. Why This Project Matters
This case study demonstrates my approach to: UX thinking and hierarchy Designing for trust-sensitive flows Government / institutional UI patterns Clean, scalable, and developer-friendly design

ESY Ration App – UX Case Study

The ESY Ration App is a mobile application concept designed to help users easily access and manage their monthly ration allotments. The goal was to create a simple, trustworthy, and user-friendly experience.

Problem Statement

Existing ration systems are often confusing due to cluttered interfaces, poor hierarchy, and unclear navigation. Users struggle to quickly find their ration details and monthly allotments.

Project Goal
  • Present essential information clearly
  • Reduce cognitive load
  • Build trust through simple design
  • Support non-technical users
Target Users
  • General public users
  • Families relying on ration services
  • Users with limited digital literacy
  • Low to mid-range smartphone users
My Role
  • UI Design
  • Basic UX planning
  • Information hierarchy
  • Visual consistency
Tools Used

Figma

Design Approach
Simplicity First

The interface focuses on minimal design, avoiding unnecessary elements and keeping interactions straightforward.

Clear Information Hierarchy

Important information such as user details and monthly allotment is placed at the top using card-based layouts.

Familiar UI Patterns

Common mobile patterns like cards, bottom navigation, and clear CTAs were used to reduce the learning curve.

Color & Accessibility

A green-based color palette was chosen to represent trust, reliability, and government-style utility applications.

Key Screens Designed
  • Home screen with user and ration details
  • Monthly allotment overview
  • Product list with quantities
  • Order tracking entry point
UX Decisions
  • Used cards instead of long text for better scanning
  • Limited actions per screen to avoid confusion
  • Maintained consistent spacing and typography
Challenges
  • Designing for users with low digital familiarity
  • Avoiding information overload
  • Balancing simplicity with completeness
Outcome

The final design delivers a clear and accessible interface suitable for MVP-level implementation and real-world use.

Learnings
  • Clear hierarchy improves usability more than visual effects
  • Simple UI decisions create better experiences
  • Designing for real users improves outcomes
Future Improvements
  • User testing with real users
  • Multi-language support
  • Accessibility enhancements
  • Responsive layouts for tablets
Conclusion

This case study highlights my ability to design practical, user-focused UI solutions with an emphasis on clarity, usability, and trust.
